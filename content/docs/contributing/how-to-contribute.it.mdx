Ci sono diversi modi per contribuire a Open Physica, a seconda del tempo che hai
a disposizione e delle tue competenze tecniche. Non preoccuparti se non sei un
esperto di programmazione: ogni contributo è prezioso!

## 1. Modifica Rapida (Via GitHub)

Il modo più semplice per correggere un errore di battitura, aggiungere una frase
mancante o fare piccole revisioni è usare l'interfaccia web di GitHub. Non serve
installare nulla sul tuo computer.

<Steps>
  <Step>
    **Vai alla pagina**: Naviga sulla pagina di Open Physica che vuoi
    modificare.
  </Step>
  <Step>
    **Apri il menu "Apri"**: Trova il pulsante "Apri" sotto il titolo della
    pagina.
  </Step>
  <Step>
    **Seleziona "GitHub"**: Questo ti porterà direttamente al file sorgente su
    GitHub.
  </Step>
  <Step>
    **Apporta le modifiche**: Usa l'editor di testo di GitHub per fare le tue
    correzioni.
  </Step>
  <Step>
    **Salva e proponi**: Clicca su "Commit changes...", scrivi una breve
    descrizione delle modifiche (es. "Corretto refuso in Elettrostatica") e
    clicca su "Propose changes".
  </Step>
  <Step>
    **Crea la Pull Request**: GitHub ti guiderà nella creazione di una "Pull
    Request" (PR), che è una richiesta agli amministratori di accettare le tue
    modifiche.
  </Step>
</Steps>

## 2. Contributo Completo (In Locale)

Se vuoi aggiungere interi capitoli, esercizi complessi o lavorare offline, è
meglio configurare l'ambiente di sviluppo. Hai due opzioni:

- **Dev Container** (consigliato) — apri il progetto in VS Code con Docker
  attivo e l'ambiente è già pronto: nessuna installazione manuale.
- **Installazione locale** — installa Bun e Git sul tuo computer.

Vai al capitolo **[Setup Ambiente](./environment-setup)** per tutti i dettagli.

## Cosa puoi contribuire?

Ecco alcune idee su cosa puoi fare:

- **Correzione di errori**: Typos, formule sbagliate, spiegazioni poco chiare.
- **Aggiunta di esercizi**: Hai un esercizio interessante? Aggiungilo alla
  sezione esercizi!
- **Miglioramento spiegazioni**: Se un paragrafo è ostico, prova a riscriverlo
  in modo più semplice.
- **Grafici e Immagini**: Aggiungi grafici interattivi o diagrammi per
  visualizzare i concetti.
- **Appunti mancanti**: Copri argomenti del corso che non sono ancora stati
  trattati.

<Callout type="tip" title="Non sai da dove iniziare?">
  Controlla la sezione "Issues" su GitHub per vedere se ci sono richieste aperte
  o task segnalati come "good first issue".
</Callout>
